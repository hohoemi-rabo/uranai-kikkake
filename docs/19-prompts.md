## 受入基準

- 3 モードすべてで `RESULT_SCHEMA` に従う JSON が返る
- charm モードの結果が「読んで嬉しくなる」トーンに揃っている
- match モードで `score: 0-100` が必ず含まれる

## 技術メモ

- プロンプトは A/B テストしやすいよう、定数として分離
- 将来的にユーザーの好みでトーン(ガチ/笑い)を切替できる構造を意識(関数化推奨)
- **`uranai-kikkake-spec.md`(Web 版仕様書)はリポジトリ外**。本チケットは REQUIREMENTS §6.2.x のヒントを元に charm と同じ「キャラ宣言 → トーン → 出力フィールド → ガード」の 4 ブロック構成で palm/match を書き起こした
- **動物例えのガードは `ANIMAL_GUARD` 共通定数**: 3 モードのプロンプト末尾に同じ文面を埋め込み。「絶滅危惧種・不気味な生物・否定的イメージ」を排除する更新は 1 箇所で済む
- **match の `score` 必須化はプロンプトの自然言語制約で担保**: `RESULT_SCHEMA.required` に `score` を入れない(charm/palm では不要)。プロンプトで「必ず含めてください」「**60 未満は絶対に出さない**」と明記してネガティブスコアも防ぐ
- **palm の「線が読めなくても架空で構わない」を明示**: 「画像が不鮮明で読めません」のような断りを Gemini が返すリスクを潰すため、自由度を文面で許可
- **否定的な内容の禁止**は palm/match にも明示。charm の「DNA 継承」だけだと弱いため「悪い手相」「破局確実」など個別 NG を列挙
